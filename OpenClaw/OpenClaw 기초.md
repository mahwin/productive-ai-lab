
## OpenClaw의 특징
- **24/7** 실행되고 있다.
- 내 **개인 PC**에서 작업이 수행된다.
- **컨텍스트**를 사용자가 소유할 뿐만 아니라 **직접 제어**할 수 있다.
- **오픈소스이기 때문에 완전히 통제**할 수 있다.
- WhatsApp, telegram, discord 같은 앱 내에서 메세지로 작업을 요청할 수 있다.
- 플러그인으로 **기능을 확장**할 수 있다.
- 방대한 커뮤니티가 지속적으로 개선하고 기능을 확장하고 있다.
- **백그라운드에서 끊임없이 동작하는 진짜 AI Agent**이다.
- LLM 모델을 자유롭게 선택할 수 있으며, 로컬 모델도 사용할 수 있다.
## OpenClaw
- 오픈클로는 채팅 앱과 AI 어이스턴트를 연결하는 **self-hosted gateway**이다.
- 외부 클라우드에 의존하지 않고 내 기기에서 24/7 동안 동작한다.
- 즉, 언제든 채팅 앱 안에서 바로 AI 어시스턴트를 사용할 수 있다.

## OpenClaw의 차별성
- self-hosted: 개인 PC나 서버에서 실행되기 때문에 모든 규칙과 설정을 통제할 수 있다.
- Multi-channel: OpenClaw는 하나의 게이트웨이 프로세스이기 때문에 여러 메시징 앱을 동시에 연결할 수 있다.
- Agent-native: 에이전트를 위해 특화되어 있기 때문에, tool use, 대화 세션 유지, longterms memory, 에이전트 간 라우팅 같은 고급 기능을 지원한다.
- Open source : MIT 라이선스로 누구나 자유롭게 사용하고, 수정하고, 기여할 수 있다.
## OpenClaw의 작업 흐름
![OpenClaw의_작업_순서도](images/OpenClaw/OpenClaw의_작업_순서도.png)
- OpenClaw는 Sessions(사용자 상태), Routing(메시지 경로), Channel connections(플랫폼과의 연결 상태)를 **하나의 Gateway로 관리**한다.
- 하나의 Gateway가 control plane 역할을 하기 때문에 **여러 채널에서 입력이 들어와도 혼란 없이 task를 처리**할 수 있다.

## OpenClaw's Dashboard
- 오픈 클로는 웹 브라우저 기반 인터페이스를 가진 **관리 콘솔을 제공**한다.
	- OpenClaw Dashboard는 Control UI나 Web UI라고도 불림
- 보안을 유지하기 위해 **로컬호스트(127.0.01)만 허용**한다. 원격으로 보고 싶다면 Tailscale이나 SSH 터널을 사용해야 한다.
- OpenClaw 시스템 전체를 한눈에 보고 조종하는 컨트롤 타워로 아래와 같은 기능을 가진다.
	1. 세션 관리: 활성 세션, 연결 상태를 확인
	2. 채널 & 채팅 관리: 연결된 메시지 앱 간 채팅 흐름을 보고 관리
	3. 구성 관리: OpenClaw의 설정이나, LLM 모델 종류, 플러그인, 메모리 설정 등을 조정
	4. 노드 관리: 여러 디바이스나 컴포넌트 연결 상태를 감독
